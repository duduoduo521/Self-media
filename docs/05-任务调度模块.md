# 任务调度模块详细设计

> 模块路径：`self-media-core/src/task` | 依赖：`self-media-ai`、`self-media-publish`、`self-media-db`、`self-media-core`(config, error)

---

## 1. 任务状态机

```
  ┌─────────┐    ┌──────────┐    ┌───────────┐
  │ Pending  │───>│ Running  │───>│ Completed │
  └─────────┘    └────┬─────┘    └───────────┘
                      │
           ┌──────────┼──────────┐
           │          │          │
           ▼          ▼          ▼
     ┌──────────┐ ┌────────┐ ┌──────────┐
     │  Failed  │ │Cancelled│ │Cancelled │
     │(可重试)  │ │(用户取消)│ │(超时取消) │
     └──────────┘ └────────┘ └──────────┘
           │
           │ (重试)
           ▼
     ┌──────────┐
     │ Running  │
     └──────────┘
```

状态转换规则：

| 从 | 到 | 触发条件 |
|----|----|---------|
| Pending | Running | 调度器开始执行 |
| Running | Completed | 所有步骤成功完成 |
| Running | Failed | 任一步骤失败且不可重试 |
| Running | Cancelled | 用户主动取消或超时 |
| Failed | Running | 重试（最多 3 次） |

## 2. 任务数据模型

### 2.1 表结构

```sql
CREATE TABLE tasks (
    id            TEXT PRIMARY KEY,
    user_id       INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    task_type     TEXT NOT NULL,          -- 'generate_publish'
    status        TEXT NOT NULL DEFAULT 'Pending',
    mode          TEXT NOT NULL,          -- 'Text' | 'Video'
    topic         TEXT NOT NULL,
    platforms     TEXT NOT NULL,          -- JSON: 目标平台列表
    progress      INTEGER NOT NULL DEFAULT 0,
    total_steps   INTEGER NOT NULL DEFAULT 0,
    current_step  TEXT,
    result        TEXT,                   -- JSON: 执行结果（发布结果等）
    error         TEXT,
    retry_count   INTEGER NOT NULL DEFAULT 0,
    max_retries   INTEGER NOT NULL DEFAULT 3,
    created_at    TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at    TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE task_steps (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id     TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    step_name   TEXT NOT NULL,
    step_order  INTEGER NOT NULL,
    status      TEXT NOT NULL DEFAULT 'Pending',
    result      TEXT,
    error       TEXT,
    started_at  TEXT,
    finished_at TEXT
);

CREATE INDEX idx_tasks_user_id ON tasks(user_id);
CREATE INDEX idx_tasks_status ON tasks(status);

-- 过期任务清理索引
CREATE INDEX idx_tasks_created_at ON tasks(created_at);
```

pub struct Task {
    pub id: String,
    pub user_id: i64,
    pub task_type: String,

    pub mode: TaskMode,
    pub topic: String,

    pub current_step: Option<String>,

    pub updated_at: String,
}

#[sqlx(type_name = "TEXT", rename_all = "snake_case")]
pub enum TaskStatus {
    Pending,
    Running,
    Completed,
    Failed,
    Cancelled,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum TaskMode {
    Text,
    Video,
}

## 3. 任务调度器

### 3.1 核心结构

```rust
pub struct TaskScheduler {
    db: SqlitePool,
    ai: MiniMaxClient,
    publishers: PublisherRegistry,
    config: ConfigService,
    concurrent_limit: usize,   // 最大并发任务数
    active_count: Arc<AtomicUsize>,  // 当前活跃任务数
    cancel_tokens: Arc<Mutex<HashMap<String, CancellationToken>>>,
}
```

### 3.2 任务创建

```rust
impl TaskScheduler {
    pub async fn create_task(
        &self,
        user_id: i64,
        mode: TaskMode,
        topic: String,
        platforms: Vec<Platform>,
    ) -> Result<Task, AppError> {
        // 1. 并发控制检查
        let active = self.active_count.load(Ordering::Relaxed);
        if active >= self.concurrent_limit {
            return Err(AppError::task(TASK_003, &format!(
                "并发任务数已达上限 ({}/{}), 请等待", active, self.concurrent_limit
            )));
        }

        // 2. 输入校验
        if topic.trim().is_empty() {
            return Err(AppError::validation(INPUT_001, "主题不能为空"));
        }
        if topic.len() > 500 {
            return Err(AppError::validation(INPUT_001, "主题长度不能超过 500 字"));
        }
        if platforms.is_empty() {
            return Err(AppError::validation(INPUT_001, "请至少选择一个发布平台"));
        }

        // 3. 生成任务步骤
        let task_id = Uuid::new_v4().to_string();
        let steps = match mode {
            TaskMode::Text => vec![
                "generate_text", "generate_images", "adapt_content", "publish"
            ],
            TaskMode::Video => vec![
                "generate_script", "generate_video", "generate_tts", "publish"
            ],
        };
        let total_steps = steps.len();

        // 4. 写入数据库
        let task = sqlx::query_as!(
            Task,
            "INSERT INTO tasks (id, user_id, task_type, status, mode, topic, platforms, total_steps) \
             VALUES (?, ?, 'generate_publish', 'Pending', ?, ?, ?, ?) RETURNING *",
            task_id, user_id, serde_json::to_string(&mode)?, topic,
            serde_json::to_string(&platforms)?, total_steps as i64
        )
        .fetch_one(&self.db)
        .await?;

        for (i, step) in steps.iter().enumerate() {
            sqlx::query!(
                "INSERT INTO task_steps (task_id, step_name, step_order) VALUES (?, ?, ?)",
                task_id, step, i as i64
            )
            .execute(&self.db)
            .await?;
        }

        // 5. 注册取消令牌
        self.cancel_tokens.lock().unwrap().insert(task_id.clone(), CancellationToken::new());

        Ok(task)
    }
```

### 3.3 任务执行

```rust
    pub async fn execute_task(&self, task_id: &str) -> Result<(), AppError> {
        let task = self.get_task(task_id).await?;

        // 检查取消令牌
        let cancel_token = self.cancel_tokens.lock().unwrap()
            .get(task_id).cloned()
            .ok_or(AppError::task(TASK_001, "任务不存在"))?;

        // 更新状态为 Running
        self.update_task_status(task_id, TaskStatus::Running).await?;
        self.active_count.fetch_add(1, Ordering::Relaxed);

        let result = match task.mode {
            TaskMode::Text => {
                self.execute_text_task(&task, &cancel_token).await
            }
            TaskMode::Video => {
                self.execute_video_task(&task, &cancel_token).await
            }
        };

        self.active_count.fetch_sub(1, Ordering::Relaxed);

        match result {
            Ok(()) => {
                self.update_task_status(task_id, TaskStatus::Completed).await?;
            }
            Err(e) => {
                // 取消任务：匹配 Task 变体中的取消标识
                let err_str = e.to_string();
                if err_str.contains("取消") {
                    self.update_task_status(task_id, TaskStatus::Cancelled).await?;
                } else {
                    // 检查是否可重试
                    let task = self.get_task(task_id).await?;
                    if task.retry_count < task.max_retries {
                        self.increment_retry(task_id).await?;
                        self.update_task_status(task_id, TaskStatus::Pending).await?;
                        tracing::warn!("任务 {} 失败，将重试 ({}/{}): {}",
                            task_id, task.retry_count + 1, task.max_retries, e);
                    } else {
                        self.update_task_error(task_id, &e.to_string()).await?;
                        self.update_task_status(task_id, TaskStatus::Failed).await?;
                    }
                }
            }
        }

        // 清理取消令牌
        self.cancel_tokens.lock().unwrap().remove(task_id);

        Ok(())
    }
```

### 3.4 文本模式执行

```rust
    async fn execute_text_task(
        &self,
        task: &Task,
        cancel_token: &CancellationToken,
    ) -> Result<(), AppError> {
        let platforms: Vec<Platform> = serde_json::from_str(&task.platforms)?;
        let platform_configs = self.config.get_platform_configs(task.user_id).await?;

        // Step 1: 生成文案
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 0, "running", None, None).await?;
        let text_req = TextRequest {
            model: "MiniMax-M2.7-highspeed".into(),
            messages: vec![Message {
                role: "user".into(),
                content: format!("请根据以下热点主题，生成一篇适合自媒体发布的文案：{}", task.topic),
            }],
            temperature: Some(0.8),
            stream: None,
        };
        let text_resp = self.ai.generate_text(text_req).await?;
        let content = text_resp.choices[0].message.content.clone();
        self.update_step(&task.id, 0, "completed", Some(&content), None).await?;

        // Step 2: 生成配图
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 1, "running", None, None).await?;
        let max_images = platform_configs.iter().map(|c| c.image_count).max().unwrap_or(3);
        let img_req = ImageRequest {
            model: "image-01".into(),
            prompt: format!("为以下内容生成配图：{}", task.topic),
            n: Some(max_images),
            aspect_ratio: Some("3:4".into()),
        };
        let img_resp = self.ai.generate_images(img_req).await?;
        let image_urls: Vec<String> = img_resp.data.iter().map(|d| d.url.clone()).collect();
        self.update_step(&task.id, 1, "completed", Some(&serde_json::to_string(&image_urls)?), None).await?;

        // Step 3: 适配各平台内容
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 2, "running", None, None).await?;
        let adapted = self.adapt_content_for_platforms(&content, &image_urls, &platform_configs).await?;
        self.update_step(&task.id, 2, "completed", Some(&serde_json::to_string(&adapted)?), None).await?;

        // Step 4: 发布
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 3, "running", None, None).await?;
        let results = self.publish_to_all(&platforms, &platform_configs, &adapted).await?;
        self.update_step(&task.id, 3, "completed", Some(&serde_json::to_string(&results)?), None).await?;

        // 更新任务进度
        sqlx::query!("UPDATE tasks SET progress = 100 WHERE id = ?", task.id)
            .execute(&self.db)
            .await?;

        Ok(())
    }
```

    async fn execute_video_task(
        &self,
        task: &Task,
        cancel_token: &CancellationToken,
    ) -> Result<(), AppError> {
        let platforms: Vec<Platform> = serde_json::from_str(&task.platforms)?;
        let platform_configs = self.config.get_platform_configs(task.user_id).await?;

        // Step 1: 生成视频脚本
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 0, "running", None, None).await?;
        let script_req = TextRequest {
            model: "MiniMax-M2.7-highspeed".into(),
            messages: vec![Message {
                role: "user".into(),
                content: format!("请为以下主题生成一个短视频脚本（30秒-1分钟）：{}", task.topic),
            }],
            temperature: Some(0.8),
            stream: None,
        };
        let script_resp = self.ai.generate_text(script_req).await?;
        let script = script_resp.choices[0].message.content.clone();
        self.update_step(&task.id, 0, "completed", Some(&script), None).await?;

        // Step 2: 生成视频
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 1, "running", None, None).await?;
        let video_req = VideoRequest {
            model: "video-01".into(),
            prompt: script.clone(),
            image: None,
        };
        let video_task = self.ai.submit_video_task(video_req).await?;
        let file_id = self.ai.poll_video_until_complete(
            &video_task.task_id,
            Duration::from_secs(1800),  // 30 分钟超时
        ).await?;
        self.update_step(&task.id, 1, "completed", Some(&file_id), None).await?;

        // Step 3: 生成配音
        cancel_token.cancelled().await?;
        self.update_step(&task.id, 2, "running", None, None).await?;
        let audio_data = self.ai.synthesize_speech(&script, "audiobook_male_1").await?;
        let audio_path = self.save_audio(&task.id, &audio_data).await?;
        self.update_step(&task.id, 2, "completed", Some(&audio_path), None).await?;

            title: task.topic.clone(),

            cover_image: None,
            tags: vec![],

        let results = self.publish_video_to_all(&platforms, &platform_configs, &video_content).await?;
        self.update_step(&task.id, 3, "completed", Some(&serde_json::to_string(&results)?), None).await?;

        sqlx::query!("UPDATE tasks SET progress = 100 WHERE id = ?", task.id)
            .execute(&self.db)

        Ok(())
    }

### 3.6 任务取消

```rust
    /// 取消任务
    pub async fn cancel_task(&self, task_id: &str) -> Result<(), AppError> {
        let task = self.get_task(task_id).await?;
        if task.status != TaskStatus::Pending && task.status != TaskStatus::Running {
            return Err(AppError::task(TASK_002, "只能取消待执行或执行中的任务"));
        }

        // 触发取消令牌
        if let Some(token) = self.cancel_tokens.lock().unwrap().get(task_id) {
            token.cancel();
        }

        self.update_task_status(task_id, TaskStatus::Cancelled).await?;
        Ok(())
    }
```

## 4. 数据清理策略

```rust
impl TaskScheduler {
    /// 清理过期数据
    pub async fn cleanup(&self) -> Result<u64, AppError> {
        // 1. 删除 30 天前的已完成/已取消/已失败任务
        let result = sqlx::query!(
            "DELETE FROM tasks WHERE status IN ('Completed', 'Cancelled', 'Failed') \
             AND created_at < datetime('now', '-30 days')"
        )
        .execute(&self.db)
        .await?;

        // 2. 清理过期会话
        sqlx::query!("DELETE FROM sessions WHERE expires_at < datetime('now')")
            .execute(&self.db)
            .await?;

        Ok(result.rows_affected())
    }
}
```

| 清理项 | 保留时长 | 执行频率 |
|--------|---------|---------|
| 已完成任务 | 30 天 | 每日 |
| 已取消任务 | 30 天 | 每日 |
| 已失败任务 | 30 天 | 每日 |
| 过期会话 | 即时 | 每日 |
