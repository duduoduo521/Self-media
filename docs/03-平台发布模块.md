# 平台发布模块详细设计

> 模块路径：`self-media-publish` | 依赖：`reqwest`、`self-media-crypto`、`self-media-core`(error)

---

## 1. 设计思路

### 1.1 核心挑战

各平台发布接口差异巨大，且均有反爬机制，主要挑战：

1. **签名/加密头**：小红书 `X-s/X-t`、抖音 `X-Bogus/a_bogus`、B站 `wbi` 等
2. **Cookie 脆弱性**：登录态频繁失效，有效期可能仅数小时
3. **API 不稳定**：各平台内部 API 无版本保证，随时变更
4. **反爬升级**：指纹检测、频率限制、验证码

### 1.2 应对策略

| 策略 | 说明 |
|------|------|
| 适配器模式 | 每个平台独立适配器 + 独立签名模块，变更互不影响 |
| 签名模块独立 | 签名逻辑放在 `sign.rs`，与发布逻辑分离，便于单独更新 |
| 健康检查 | 定时检测 Cookie 有效性，提前预警 |
| 降级方案 | 对于签名失效的平台，可降级到 headless browser 方案 |
| 频率控制 | 每个平台独立限速，避免触发反爬 |

## 2. 适配器 Trait 设计

> **注意**：`Platform`、`HotspotSource` 等共享枚举定义在 `self-media-core/src/types.rs` 中，`self-media-publish` 通过依赖 `self-media-core` 引用，避免循环依赖。

```rust
#[async_trait]
pub trait PlatformPublisher: Send + Sync {
    /// 平台标识
    fn platform(&self) -> Platform;

    /// 检查登录态是否有效
    async fn check_login_status(&self, credential: &PlatformCredential) -> Result<bool, AppError>;

    /// 触发登录流程（大多数平台需前端扫码）
    async fn login(&self, credential: &PlatformCredential) -> Result<PlatformCredential, AppError>;

    /// 发布图文/文章
    async fn publish_article(
        &self,
        credential: &PlatformCredential,
        content: &ArticleContent,
    ) -> Result<PublishResult, AppError>;

    /// 发布视频
    async fn publish_video(
        &self,
        credential: &PlatformCredential,
        content: &VideoContent,
    ) -> Result<PublishResult, AppError>;

    /// 上传图片
    async fn upload_image(
        &self,
        credential: &PlatformCredential,
        image_data: &[u8],
        filename: &str,
    ) -> Result<String, AppError>;

    /// 获取平台发布频率限制（返回最小间隔秒数）
    fn rate_limit_seconds(&self) -> u64 {
        30  // 默认 30 秒间隔
    }
}
```

```rust
impl PublisherRegistry {
    pub fn new(http_client: Client) -> Self {
        let mut publishers: HashMap<Platform, Box<dyn PlatformPublisher>> = HashMap::new();
        publishers.insert(Platform::Xiaohongshu, Box::new(XiaohongshuPublisher::new(http_client.clone())));
        publishers.insert(Platform::Douyin, Box::new(DouyinPublisher::new(http_client.clone())));
        publishers.insert(Platform::WeChatOfficial, Box::new(WeChatPublisher::new(http_client.clone())));
        publishers.insert(Platform::Bilibili, Box::new(BilibiliPublisher::new(http_client.clone())));
        publishers.insert(Platform::Weibo, Box::new(WeiboPublisher::new(http_client.clone())));
        publishers.insert(Platform::Toutiao, Box::new(ToutiaoPublisher::new(http_client.clone())));
        Self { publishers }
    }

    pub fn get(&self, platform: &Platform) -> Result<&dyn PlatformPublisher, AppError> {
        self.publishers
            .get(platform)
            .map(|p| p.as_ref())
            .ok_or(AppError::platform(PLAT_004, &format!("不支持的平台: {:?}", platform)))
    }

    pub fn all_platforms(&self) -> Vec<Platform> {
        self.publishers.keys().cloned().collect()
    }
}
```

### 4.2 发布适配器 (`publisher.rs`)

```rust
pub struct XiaohongshuPublisher {
    http: Client,
}

impl XiaohongshuPublisher {
    pub fn new(http: Client) -> Self {
        Self { http }
    }

    const NOTE_URL: &'static str = "https://edith.xiaohongshu.com/api/sns/web/v1/feed";
    const UPLOAD_URL: &'static str = "https://edith.xiaohongshu.com/api/sns/web/v1/upload_image";
    const USER_INFO_URL: &'static str = "https://edith.xiaohongshu.com/api/sns/web/v1/user/selfinfo";
}

#[async_trait]
impl PlatformPublisher for XiaohongshuPublisher {
    fn platform(&self) -> Platform {
        Platform::Xiaohongshu
    }

    fn rate_limit_seconds(&self) -> u64 {
        60  // 小红书发布间隔至少 60 秒
    }

    async fn check_login_status(&self, credential: &PlatformCredential) -> Result<bool, AppError> {
        let (x_s, x_t) = XiaohongshuSigner::sign(Self::USER_INFO_URL, None)?;
        let resp = self.http
            .get(Self::USER_INFO_URL)
            .header("Cookie", &credential.cookies)
            .header("X-s", x_s)
            .header("X-t", x_t)
            .header("User-Agent", "Mozilla/5.0 ...")  // 需模拟浏览器 UA
            .send()
            .await?;
        Ok(resp.status().is_success())
    }

    async fn login(&self, _credential: &PlatformCredential) -> Result<PlatformCredential, AppError> {
        Err(AppError::platform(PLAT_001, "小红书需扫码登录，请在前端完成"))
    }

    async fn publish_article(
        &self,
        credential: &PlatformCredential,
        content: &ArticleContent,
    ) -> Result<PublishResult, AppError> {
        // Step 1: 上传图片
        let mut image_ids = Vec::new();
        for url in &content.image_urls {
            let img_data = self.http.get(url).send().await?.bytes().await?;
            let img_id = self.upload_image(credential, &img_data, "image.png").await?;
            image_ids.push(img_id);
        }

        // Step 2: 构建请求体
        let body = serde_json::json!({
            "title": content.title,
            "desc": content.body,
            "image_info": {
                "image_list": image_ids.iter().map(|id| serde_json::json!({"image_id": id})).collect::<Vec<_>>()
            },
            "tag_list": content.tags,
            "topic_id": content.topic,
        });

        // Step 3: 签名并发送
        let body_str = serde_json::to_string(&body)?;
        let (x_s, x_t) = XiaohongshuSigner::sign(Self::NOTE_URL, Some(&body_str))?;

        let resp = self.http
            .post(Self::NOTE_URL)
            .header("Cookie", &credential.cookies)
            .header("X-s", x_s)
            .header("X-t", x_t)
            .header("Content-Type", "application/json")
            .header("User-Agent", "Mozilla/5.0 ...")
            .json(&body)
            .send()
            .await?;

        if resp.status().is_success() {
            let result: serde_json::Value = resp.json().await?;
            Ok(PublishResult {
                platform: Platform::Xiaohongshu,
                success: true,
                post_id: result["note_id"].as_str().map(|s| s.to_string()),
                post_url: None,
                error_code: None,
                error_message: None,
            })
        } else {
            let status = resp.status();
            let error_body = resp.text().await.unwrap_or_default();
            Ok(PublishResult {
                platform: Platform::Xiaohongshu,
                success: false,
                post_id: None,
                post_url: None,
                error_code: Some(format!("HTTP_{}", status.as_u16())),
                error_message: Some(format!("发布失败: HTTP {} - {}", status, error_body)),
            })
        }
    }

    async fn publish_video(
        &self,
        credential: &PlatformCredential,
        content: &VideoContent,
    ) -> Result<PublishResult, AppError> {

        // 2. 获取 video_id
        // 3. 发布笔记携带视频

    }

    async fn upload_image(
        &self,
        credential: &PlatformCredential,
        image_data: &[u8],
        filename: &str,
    ) -> Result<String, AppError> {
        let (x_s, x_t) = XiaohongshuSigner::sign(Self::UPLOAD_URL, None)?;

        let part = Part::bytes(image_data.to_vec())
            .file_name(filename.to_string())
            .mime_str("image/png")?;

        let form = Form::new()

        let resp = self.http

            .header("X-s", x_s)
            .header("X-t", x_t)

            .multipart(form)
            .send()
            .await?;

        let result: serde_json::Value = resp.json().await?;
        let image_id = result["image_id"].as_str()
            .ok_or(AppError::platform(PLAT_002, "图片上传失败：未获取到 image_id"))?;
        Ok(image_id.to_string())
```

## 5. 各平台签名要求

| 平台 | 签名头 | 签名算法 | 维护难度 |
|------|--------|---------|---------|
| 小红书 | `X-s`, `X-t` | 基于 JS 混淆逻辑 | 高 |
| 抖音 | `X-Bogus`, `a_bogus` | 基于 JS 混淆逻辑 | 高 |
| B站 | `wbi` 签名 | 基于公钥的 URL 参数签名 | 中 |
| 微博 | 无特殊签名 | Cookie 即可 | 低 |
| 头条 | `X-Tt-Token` | Cookie 中的 token | 低 |
| 公众号 | 官方 API + access_token | OAuth2 流程 | 低（官方接口） |

### 5.1 签名模块设计原则

1. **独立文件**：每个平台的签名逻辑放在独立 `sign.rs` 中
2. **JS 引擎优先**：推荐使用 `boa_engine` 执行平台原始 JS，减少逆向维护成本
3. **外部签名服务备选**：可将签名逻辑部署为独立微服务，方便热更新
4. **降级处理**：签名失败时返回明确错误码 `PLAT_004`，提示用户平台 API 可能变更

## 6. Cookie 健康检查

```rust
/// 定期检查所有平台 Cookie 有效性
pub struct CookieHealthChecker {
    registry: PublisherRegistry,
    db: SqlitePool,
    user_key_cache: UserKeyCache,   // 多用户密钥缓存
}

impl CookieHealthChecker {
    /// 检查单个平台
    pub async fn check_platform(&self, user_id: i64, platform: &Platform) -> Result<CookieStatus, AppError> {
        let config = self.get_platform_config(user_id, platform).await?;

        // 从缓存获取对应用户的 UserKey
        let user_key = self.user_key_cache.get_or_derive(user_id, &self.db, None).await?;

        let credential = PlatformCredential {
            platform: platform.clone(),
            cookies: user_key.decrypt(&config.cookies.unwrap_or_default())?,
            extra: HashMap::new(),
        };

        let publisher = self.registry.get(platform)?;
        let valid = publisher.check_login_status(&credential).await;

        Ok(CookieStatus {
            platform: platform.clone(),
            valid: valid.unwrap_or(false),
            last_checked: Utc::now(),
        })
    }

    /// 检查用户所有已启用平台
    pub async fn check_all(&self, user_id: i64) -> Result<Vec<CookieStatus>, AppError> {
        let configs = self.get_enabled_platforms(user_id).await?;
        let mut results = Vec::new();
        for config in configs {
            let status = self.check_platform(user_id, &config.platform).await?;
            results.push(status);
        }
        Ok(results)
    }
}

#[derive(Debug, Serialize, Deserialize)]
pub struct CookieStatus {
    pub platform: Platform,
    pub valid: bool,
    pub last_checked: DateTime<Utc>,
}
```

## 7. 发布频率控制

```rust
use std::sync::Mutex;
use std::collections::HashMap;

/// 发布频率控制器
pub struct RateLimiter {
    last_publish: Mutex<HashMap<Platform, Instant>>,
}

impl RateLimiter {
    pub fn new() -> Self {
        Self {
            last_publish: Mutex::new(HashMap::new()),
        }
    }

    /// 检查是否可以发布，如不可则等待
    pub async fn acquire(&self, platform: &Platform, min_interval_secs: u64) -> Result<(), AppError> {
        let mut map = self.last_publish.lock().unwrap();
        let now = Instant::now();
        if let Some(last) = map.get(platform) {
            let elapsed = now.duration_since(*last);
            if elapsed < Duration::from_secs(min_interval_secs) {
                let wait = Duration::from_secs(min_interval_secs) - elapsed;
                drop(map); // 释放锁
                tokio::time::sleep(wait).await;
                let mut map = self.last_publish.lock().unwrap();
                map.insert(platform.clone(), Instant::now());
            } else {
                map.insert(platform.clone(), now);
            }
        } else {
            map.insert(platform.clone(), now);
        }
        Ok(())
    }
}
```
