# 构建与部署详细文档

---

## 1. 开发环境

### 1.1 前置依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| Rust | 1.75+ | 后端编译 |
| Node.js | 18+ | 前端构建 |
| pnpm | 8+ | 前端包管理 |
| Tauri CLI | 2.x | 桌面端开发/打包 |

### 1.2 开发命令

```bash
# 安装依赖
cargo build
cd frontend && pnpm install

# 桌面端开发（Tauri）
cd crates/self-media-app && cargo tauri dev

# Web 端开发（需两个终端）
cargo run -p self-media-web    # 终端1：Rust 后端 (默认 :3000)
cd frontend && pnpm dev         # 终端2：Vue 前端 (默认 :5173)
```

### 1.3 环境变量

```bash
# .env (开发环境)
SELF_MEDIA_MACHINE_KEY=dev-machine-key-change-in-production
SELF_MEDIA_DB_PATH=./data/self-media.db
SELF_MEDIA_WEB_PORT=3000
SELF_MEDIA_LOG_LEVEL=debug
```

## 2. 生产构建

### 2.1 桌面端打包

```bash
cd crates/self-media-app && cargo tauri build
```

输出：

| 平台 | 产物 | 预估体积 |
|------|------|---------|
| Windows | `target/release/bundle/msi/SelfMedia_1.0.0_x64.msi` | ~12MB |
| macOS | `target/release/bundle/dmg/SelfMedia_1.0.0_x64.dmg` | ~15MB |

### 2.2 Web 端打包

```bash
# 前端构建
cd frontend && pnpm build

# 后端构建
cargo build -p self-media-web --release
```

输出：
- `self-media-web` 二进制
- `frontend/dist/` 静态文件

部署时将 `frontend/dist/` 托管为静态资源，或由 Axum 直接 serve。

## 3. 数据库迁移

使用 `sqlx-cli` 管理：

```bash
# 安装 sqlx-cli
cargo install sqlx-cli --features sqlite

# 创建迁移
sqlx migrate add create_users

# 执行迁移
sqlx migrate run

# 回滚
sqlx migrate revert
```

迁移文件统一放在 `migrations/` 目录，文档中各模块的 DDL 需同步到对应迁移文件中：

```
migrations/
├── 001_create_users.sql
├── 002_create_sessions.sql
├── 003_create_platform_configs.sql
├── 004_create_api_keys.sql
├── 005_create_user_preferences.sql
├── 006_create_tasks.sql
└── 007_create_task_steps.sql
```

## 4. 日志规范

### 4.1 日志级别

| 级别 | 使用场景 |
|------|---------|
| ERROR | 不可恢复错误（数据库连接失败、加密失败） |
| WARN | 可恢复异常（单源热点获取失败、Cookie 过期） |
| INFO | 关键业务操作（用户注册/登录、任务创建/完成、发布成功/失败） |
| DEBUG | 请求/响应细节（仅开发环境） |
| TRACE | 性能追踪（仅开发环境） |

### 4.2 日志格式

```
2026-04-09T10:30:00.123Z INFO  self_media_core::task::scheduler: task completed task_id=abc-123 mode=Text duration=45s
2026-04-09T10:30:01.456Z WARN  self_media_core::hotspot::service: source fetch failed source=Zhihu error="Connection timeout"
2026-04-09T10:30:02.789Z ERROR self_media_core::user::service: login failed username=t***t error="password mismatch"
```

### 4.3 关键审计日志

以下操作必须记录 INFO 级别日志：

- 用户注册、登录、登出、密码修改
- API Key 设置
- 任务创建、执行、取消、完成、失败
- 平台发布成功/失败
- Cookie 健康检查结果

## 6. 合规提示

本工具定位为**辅助创作者提高效率**，用户需注意：

1. **遵守各平台用户协议**：部分平台禁止使用自动化工具发布内容
2. **控制发布频率**：建议每日同一平台发布不超过 3 条
3. **内容质量**：AI 生成内容需人工审核后发布，避免低质/违规内容
4. **账号安全**：频繁自动化操作可能导致账号被风控

应用内应在首次使用时弹出合规提示，用户确认后方可继续使用。
