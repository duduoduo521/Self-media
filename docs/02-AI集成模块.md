# AI 集成模块详细设计

> 模块路径：`self-media-ai` | 依赖：`reqwest`、`self-media-core`(error)

---

## 1. 设计思路

**不捆绑 CLI 二进制**，直接通过 reqwest 调用 MiniMax REST API。优势：

- 消除 Node.js 运行时依赖，二进制体积减少 50MB+
- API 调用延迟更低（无子进程启动开销）
- 错误处理更精细（HTTP 状态码 + 响应体解析）
- 支持流式输出（SSE）用于实时文本生成
- 支持异步任务轮询（视频生成）

---

## 2. API 端点映射

| 功能 | MiniMax API 端点 | 方法 | 备注 |
|------|-----------------|------|------|
| 文本生成 | `/v1/text/chatcompletion_v2` | POST | 支持 SSE 流式 |
| 图片生成 | `/v1/image/generation` | POST | 同步返回 |
| 视频生成（提交） | `/v1/video_generation` | POST | 异步任务 |
| 视频生成（查询） | `/v1/query/video_generation?task_id={id}` | GET | 轮询 |
| 视频下载 | `/v1/files/retrieval?file_id={id}` | GET | 返回下载 URL |
| 语音合成 | `/v1/t2a_v2` | POST | 同步返回音频 |
| 异步长文本语音 | `/v1/t2a_async_v2` | POST | 异步任务 |

## 3. 客户端设计

### 3.1 基础结构

```rust
pub struct MiniMaxClient {
    http: Client,
    api_key: String,
    base_url: String,
}

pub enum MiniMaxRegion {
    CN,
    Global,
}

impl MiniMaxRegion {
    pub fn base_url(&self) -> &'static str {
        match self {
            Self::CN => "https://api.minimaxi.com",
            Self::Global => "https://api.minimax.io",
        }
    }
}

impl MiniMaxClient {
    pub fn new(api_key: String, region: MiniMaxRegion) -> Self {
        let base_url = region.base_url().to_string();
        let http = Client::builder()
            .timeout(Duration::from_secs(120))
            .build()
            .expect("HTTP client build failed");
        Self { http, api_key, base_url }
    }

    /// 构建带认证的请求
    fn authenticated_request(&self, method: Method, path: &str) -> RequestBuilder {
        self.http
            .request(method, format!("{}{}", self.base_url, path))
            .header("Authorization", format!("Bearer {}", self.api_key))
    }
}
```

> **修复说明**：移除冗余的 `region` 字段，构造时即计算 `base_url`；提取 `authenticated_request` 消除重复代码。

## 4. 核心方法实现

### 4.1 文本生成（含 SSE 流式）

```rust
impl MiniMaxClient {
    /// 非流式文本生成
    pub async fn generate_text(&self, req: TextRequest) -> Result<TextResponse, AppError> {
        let resp = self.authenticated_request(Method::POST, "/v1/text/chatcompletion_v2")
            .json(&req)
            .send()
            .await?
            .error_for_status()
            .map_err(|e| self.map_api_error(e))?
            .json::<TextResponse>()
            .await?;
        Ok(resp)
    }

    /// 流式文本生成（SSE）
    /// 返回 Stream，调用方逐 chunk 处理
    pub async fn generate_text_stream(
        &self,
        req: TextRequest,
    ) -> Result<impl Stream<Item = Result<TextChunk, AppError>>, AppError> {
        let mut stream_req = req;
        stream_req.stream = Some(true);

        let resp = self.authenticated_request(Method::POST, "/v1/text/chatcompletion_v2")
            .json(&stream_req)
            .send()
            .await?
            .error_for_status()
            .map_err(|e| self.map_api_error(e))?;

        Ok(parse_sse_stream(resp.bytes_stream()))
    }
}

/// SSE 流解析器
/// 处理 TCP 分包导致的数据切分：使用缓冲区累积数据，按 "\n\n" 分割 SSE 事件块
fn parse_sse_stream(
    byte_stream: impl Stream<Item = Result<Bytes, reqwest::Error>>,
) -> impl Stream<Item = Result<TextChunk, AppError>> {
    futures::stream::unfold(
        (byte_stream, String::new()),
        |(byte_stream, mut buffer)| async move {
            loop {
                // 尝试从缓冲区解析完整的 SSE 事件
                if let Some(pos) = buffer.find("\n\n") {
                    let block = buffer[..pos].to_string();
                    buffer.drain(..pos + 2);

                    for line in block.lines() {
                        if let Some(data) = line.strip_prefix("data: ") {
                            if data == "[DONE]" {
                                return None;  // 流结束
                            }
                            if let Ok(chunk) = serde_json::from_str::<TextChunk>(data) {
                                return Some((Ok(chunk), (byte_stream, buffer)));
                            }
                        }
                    }
                    // 当前块无有效 data 行，继续读下一个块
                    continue;
                }

                // 缓冲区中没有完整事件，读取更多数据
                use futures::StreamExt;
                let (item, remaining_stream) = byte_stream.into_future().await;
                match item {
                    Some(Ok(bytes)) => {
                        buffer.push_str(&String::from_utf8_lossy(&bytes));
                        continue;  // 重新尝试解析
                    }
                    Some(Err(e)) => {
                        return Some((Err(AppError::ai(AI_002, &format!("流读取失败: {}", e))), (remaining_stream, buffer)));
                    }
                    None => return None,  // 流结束
                }
            }
        },
    )
}

    /// 语音合成
    /// MiniMax TTS API 返回 JSON（包含 data.hex），需解析后转音频字节
    pub async fn synthesize_speech(&self, text: &str, voice_id: &str) -> Result<Vec<u8>, AppError> {
        let req_body = SpeechRequest {
            model: "speech-02-hd".into(),
            text: text.into(),
            stream: false,
            voice_setting: VoiceSetting {
                voice_id: voice_id.into(),
                speed: 1.0,
            },
            audio_setting: AudioSetting {
                sample_rate: 32000,
                format: "mp3".into(),
                channel: 1,
            },
        };

        let resp = self.authenticated_request(Method::POST, "/v1/t2a_v2")
            .json(&req_body)
            .send()
            .await?
            .error_for_status()
            .map_err(|e| self.map_api_error(e))?
            .json::<SpeechResponse>()
            .await?;

        // 从 hex 编码解码为字节
        let audio_bytes = hex::decode(&resp.data.hex)
            .map_err(|e| AppError::ai(AI_002, &format!("音频解码失败: {}", e)))?;
        Ok(audio_bytes)
    }

    /// API 错误映射
    fn map_api_error(&self, e: reqwest::Error) -> AppError {
        match e.status() {
            Some(StatusCode::UNAUTHORIZED) | Some(StatusCode::FORBIDDEN) => {
                AppError::ai(AI_001, "API Key 无效或权限不足")
            }
            Some(StatusCode::TOO_MANY_REQUESTS) => {
                AppError::ai(AI_003, "API 调用配额不足或限流")
            }
            Some(StatusCode::REQUEST_TIMEOUT) | Some(StatusCode::GATEWAY_TIMEOUT) => {
                AppError::ai(AI_002, "AI 生成超时")
            }
            _ => AppError::ai(AI_002, &format!("AI 服务错误: {}", e)),
        }
    }
}
```

### 4.2 视频任务轮询（带超时和退避）

```rust
impl MiniMaxClient {
    /// 轮询视频任务直到完成，带超时和指数退避
    pub async fn poll_video_until_complete(
        &self,
        task_id: &str,
        max_duration: Duration,  // 最大等待时长，建议 30 分钟
    ) -> Result<String, AppError> {
        let start = Instant::now();
        let mut interval = Duration::from_secs(5);   // 初始轮询间隔
        let max_interval = Duration::from_secs(30);   // 最大轮询间隔

        loop {
            if start.elapsed() > max_duration {
                return Err(AppError::task(TASK_002, "视频生成超时，已自动取消"));
            }

            tokio::time::sleep(interval).await;

            let status = self.poll_video_task(task_id).await?;
            match status.status.as_str() {
                "Success" => {
                    return status.file_id.ok_or(AppError::ai(AI_002, "视频生成成功但未返回文件 ID"));
                }
                "Failed" => {
                    let msg = status.error.unwrap_or_default();
                    return Err(AppError::ai(AI_002, &format!("视频生成失败: {}", msg)));
                }
                _ => {
                    // Pending / Running：指数退避
                    interval = (interval * 2).min(max_interval);
                }
            }
        }
    }
}
```

> **修复说明**：增加最大等待时长（默认 30 分钟），指数退避（5s → 10s → 20s → 30s），防止无限轮询。
